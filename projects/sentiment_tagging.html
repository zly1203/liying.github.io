SentiFlow: Reclaiming Efficiency in
Social Listening Workflows.

Inaccurate auto-tagging in commercial tools (Talkwalker).

Custom AI Layer + Human-in-the-Loop Audit.

Reduced manual audit time & improved data quality.

The Context

Commercial social listening tools like Talkwalker are great for data collection, but their out-of-the-box sentiment models often fail to grasp brand-specific nuances or sarcasm. In practice, this leads to hours of manual re-tagging by analysts to ensure report accuracy.

SentiFlow was built as a bridge. It adds a customizable AI layer to pre-tag data and, more importantly, flags exactly which rows the machine is "unsure" about.

[ STREAMLIT_INTERFACE_DASHBOARD ]

How It Works

Instead of blind automation, I implemented a "Confidence Gate." The workflow automatically accepts high-probability tags and isolates low-confidence ones for an analyst to review via a Streamlit interface. This ensures we aren't just faster, but also more reliable.

# Filtering for human attention
threshold = 0.85
needs_review = df[df['confidence'] < threshold]
auto_pass = df[df['confidence'] >= threshold]
"We don't need the AI to be perfect; we need it to know when it's confused. That's where the real efficiency gain happens."

Practical Outcomes

The system moved the team away from "tagging everything" to "auditing the exceptions." This significantly reduced the time spent on data cleanup for monthly reports while building a cleaner, brand-specific dataset that can be used for future model fine-tuning.
